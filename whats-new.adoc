== 2026 年 5 月 3 日



=== Amazon FSX for NetApp ONTAP 產品



==== 區塊裝置複本

Workload Factory 為您的區塊裝置提供複製功能，以便您可以快速佈建可寫入且高效的資料副本，同時控制容量、存取和生命週期。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["在 NetApp Workload Factory 中管理區塊儲存"]



==== 區塊裝置上的 EC2 標籤

區塊裝置表格現在包含一欄，顯示正在使用每個區塊裝置的 EC2 主機標籤。這有助於您在疑難排解或稽核期間識別哪個主機使用每個區塊裝置。

若要在每個區塊裝置上建立標籤，請使用我們提供的使用者腳本部署 EC2 執行個體。

link:https://github.com/NetApp/FSx-ONTAP-samples-scripts/tree/main/Management-Utilities/ec2-user-data-iscsi-create-and-mount["EC2 使用者資料指令碼，用於將標籤指派給區塊裝置"]



==== FSx for ONTAP 檔案系統的容量趨勢

直接從 FSx for ONTAP 檔案系統的概覽中了解 SSD 容量趨勢。新增的容量管理卡顯示了上個月的使用趨勢。此分析比較了實際容量和分配容量，方便您評估資源使用量。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-capacity-automatically.html["在 NetApp Workload Factory 中自動管理容量"]



==== FSx for ONTAP 檔案系統架構良好狀態的 PDF 報告

分析完成後，您可以下載一份 PDF 報告，其中包含您的 FSx for ONTAP 檔案系統架構良好狀態的資訊，供您存檔或與您的團隊分享。link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["在 NetApp Workload Factory 中查看 FSx for ONTAP 檔案系統的架構良好狀態。"]



=== 設定與管理



==== Workload Factory 的電子郵件通知

現在，您無需設定 Amazon SNS 主題即可透過電子郵件接收 Workload Factory 通知。啟用電子郵件通知功能，即可傳送通知至與該帳戶關聯的任何電子郵件地址。

您可以透過 Workload Factory 控制台或 NetApp Console 設定電子郵件通知。NetApp Console 還提供配置選項，可讓您設定顯示在鈴鐺圖示下的 NetApp Console 通知。

如果在 NetApp Console 中配置通知，NetApp Console 將負責傳送通知。Workload Factory 主控台中的通知設定將變為唯讀，以防止重複通知。



==== EDA 延遲的新通知

NetApp Workload Factory 通知服務包含 EDA 延遲通知，當 EDA 操作的延遲超過一定閾值時，該通知會通知您。

link:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["設定 Workload Factory 通知"]



==== FSx for ONTAP 檔案系統的新通知

NetApp Workload Factory 通知服務包括每月向您的帳戶發送成本摘要通知，其中顯示了支出趨勢、異常變化以及 FSx for ONTAP 檔案系統的最佳節省機會。

link:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["設定 Workload Factory 通知"]



=== EDA 工作負載



==== 延遲事件的電子郵件和 SNS 通知

現在，當 FSx for ONTAP 磁碟區上偵測到延遲事件時，您可以接收自動通知。在 Workload Factory 設定中設定電子郵件或 Amazon SNS 通知，以便在磁碟區的延遲超過您設定的閾值時接收即時警報。

通知會以檔案系統為基礎傳送，包括所有受影響磁碟區的詳細資訊、嚴重性層級、臨界值比較，以及可直接連結至 Latency 頁面進行調查的連結。

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#configure-latency-notifications["了解如何設定延遲通知"^]。



==== 使用互動式圖表將延遲趨勢視覺化

延遲分析面板現在包含分頁式介面，可進行全面的延遲分析：

* *概述*：檢視自動化基本分析結果，顯示哪個元件造成延遲
* *隨時間變化*：透過互動式圖表視覺化多個時間範圍（1 小時、3 小時、12 小時、24 小時或 72 小時）內的磁碟區延遲行為


*Over time* 標籤顯示原始 CloudWatch 延遲資料（讀取或寫入，取決於觸發的警示），以及警告和嚴重臨界值線、違規指標和每次違規的詳細指標。

這種全面的視覺化功能可協助您識別模式、瞭解延遲問題是反覆出現還是孤立發生，並針對補救措施做出更明智的決策。

https://docs.netapp.com/us-en/workload-eda/task-analyze-latency-trends.html["瞭解如何分析延遲趨勢"^]。



== 2026 年 4 月 6 日



=== Amazon FSX for NetApp ONTAP 產品



==== 精心設計的分析更新

Workload Factory 會分析您的 FSx for ONTAP 檔案系統，尋找*非使用中的 NAS 磁碟區*。組態分析會識別未主動使用的 NAS 磁碟區，並建議採取行動以最佳化儲存使用率並降低成本。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["為 ONTAP 檔案系統實作架構良好的 FSx"]



==== S3 存取點的日誌表功能

日誌表功能提供了一種大規模管理 FSx ONTAP S3 物件儲存的方法，可讓您輕鬆稽核使用者對儲存貯體中物件的存取事件和操作。使用 Workload Factory，您可以設定儲存貯體和 S3 表的自動掃描，以記錄事件，顯示所有物件、其詳細資訊和使用者操作。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/setup-journal-table.html["為 NetApp Workload Factory 設定日誌表格基礎架構"]



==== 支援管理現有 igroup

Workload Factory 支援管理現有的 igroup。您可以新增或移除區塊裝置和主機啟動器、以準時提供已部署的工作負載。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["在 NetApp Workload Factory 中管理區塊儲存"]



==== 支援指定 FlexVol volume 的 Aggregate

Workload Factory 支援在選定的 Aggregate 上建立 FlexVol 磁碟區。這樣，您可以將容量和效能負載分散到所有檔案系統資源。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-volume.html["在 NetApp Workload Factory 中建立 FlexVol volume"]



==== 區塊裝置的最後存取時間

Workload Factory 會回報區塊裝置的最後存取時間。該時間是根據您上次在 Workload Factory 中存取 igroup 的時間。了解每個區塊裝置的最後使用時間有助於您優化資源。



=== 資料庫工作負載



==== 針對 Oracle 的良好架構分析

架構完善的分析包括以下針對 Oracle 的新組態：

* *Oracle Critical Patch Updates*：評估支援的 Oracle 自我管理資料庫安裝中缺少的安全性修補程式。
* *跨區域複寫*：檢查是否為服務於 Oracle 資料庫的 FSx for ONTAP 檔案系統啟用了跨區域複寫（CRR）。
* *應用程式一致性快照*：檢查 Oracle 資料庫是否受應用程式一致性快照保護，並使用 NetApp SnapCenter。


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["在 Workload Factory 中實作架構良好的資料庫配置"]



==== Oracle 的一次性評估

Workload Factory 現在提供一次性評估，可隨時分析您的 Oracle 資料庫部署配置 - 新部署、遷移後、故障排除等。您將獲得有助於最佳化部署配置並遵循最佳實務的建議。這是您無需完全投入設定流程即可使用架構完善功能的機會。您可以在不於 Workload Factory 中儲存憑證、不註冊資料庫資源或不設定外部連線的情況下評估資料庫。

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["對您的 Oracle 資料庫部署進行一次性評估"]



==== 改善多個 Oracle 資料庫的成本節約

我們增強了 Oracle 的成本節約分析，為多個 Oracle 內部部署資料庫提供統一的儲存解決方案，這些資料庫將儲存在一個 FSx for ONTAP 檔案系統上。這種整合透過減少多個資料庫所需的檔案系統數量來優化儲存成本，從而進一步降低成本。

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["探索在內部部署 Oracle 上偵測到的資料庫的節省方案"]

=== EDA 工作負載



==== EDA 容量建議

Workload Factory 現在可以自動分析 EDA 清單中每個 FSx for ONTAP 叢集的 SSD 容量，並在需要調整時產生容量建議。該演算法每 24 小時對每個叢集執行一次。

當識別出建議時：

* 您將立即收到電子郵件或 WAD 通知（不包含在每週摘要中）。
* 在「叢集」表格中，受影響的叢集上會顯示燈泡指示器。
* Overview 儀表板上的摘要顯示具有作用中建議的叢集總數。


選取具有作用中建議的叢集以檢視建議，其中說明建議的變更及其原因。

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#capacity-recommendations["深入瞭解容量建議"^]。



==== 自動化 AI 延遲分析

延遲分析現在包括 AI 代理分析。

對於涉及資料或叢集延遲的複雜案例、您可以叫用 AI 代理程式分析以接收：

* 詳細的根本原因識別
* 受影響的 EC2 用戶端清單
* 具體補救步驟


AI 代理分析需要 Amazon Bedrock 模型 ARN 配置，並可按需提供，適用於基礎分析建議進行更深入調查的場景。AI 代理可以識別諸如 bully 磁碟區、掛載點配置不佳、FlexGroup 需要重新平衡以及橫向擴展需求等問題。

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#run-ai-agent-analysis["深入瞭解 AI 代理程式分析"^]。

=== VMware 工作負載



==== 為 Google Cloud VMware Engine 引入遷移顧問支援

NetApp Workload Factory 現在支援 Google Cloud VMware Engine。您可以將本地 VMware 工作負載遷移到 Google Cloud VMware Engine，並使用 Google Cloud NetApp Volumes 作為外部 NFS 資料存儲，從而優化成本並更好地控制 VMware 環境，而無需重構或重新平台化應用程式。

Google Cloud VMware Engine 遷移顧問提供：

* 支援 VE1 和 VE2 節點類型的自動化 Google Cloud VMware Engine 叢集容量規劃和設計
* vSAN 內部資料儲存與 Google Cloud NetApp Volumes 外部 NFS 資料儲存之間的 VM 佈局最佳化。
* 對僅使用 vSAN 的配置和使用 Google Cloud NetApp Volumes 增強的配置進行詳細的成本比較和 TCO 分析。
* 支援標準版、進階版和至尊版 Google Cloud NetApp Volumes 服務等級。
* 使用 Terraform 進行基礎架構即程式碼部署，實現自動化配置。

=== 資料庫工作負載



==== SQL Server 一次性評估

Workload Factory 現在提供一次性評估，可隨時分析您的 SQL Server 資料庫部署配置 - 新部署、遷移後、故障排除等。您將獲得有助於最佳化部署配置並遵循最佳實務的建議。這是您無需完全投入設定流程即可使用架構完善功能的機會。您可以在不於 Workload Factory 中儲存憑證、不註冊資料庫資源或不設定外部連線的情況下評估資料庫。

==== 支援在 Microsoft SQL Server 上註冊 Always On 可用性群組，以及在 Oracle 上註冊 Data Guard

現在，您可以在 NetApp Workload Factory 的資料庫工作負載中發現並註冊採用 Always On 可用性群組（AOAG）部署模型的 Microsoft SQL Server 執行個體以及採用 Data Guard 部署模型的 Oracle 資料庫。此增強功能可讓您在 Workload Factory 主控台中更好地管理 SQL Server 和 Oracle 資源。

註冊 AOAG 執行個體和 Data Guard 資料庫後、您可以利用 Workload Factory for Databases 中的各種功能、例如檢視資源詳細資料和分析錯誤記錄。

Workload Factory for Databases 也支援採用 AOAG 部署模型的 Microsoft SQL Server 執行個體進行架構完善的分析。您可以分析可用性群組中 SQL Server 執行個體的組態，並獲得最佳化部署組態和遵循最佳實務做法的建議。註冊您的 AOAG 執行個體即可啟用此功能。

link:https://docs.netapp.com/us-en/workload-databases/register-instance.html#register-a-sql-server-instance["在 Workload Factory for Databases 註冊資源"]



==== 針對 Oracle 的良好架構分析

精心設計的分析包括 Oracle 的 *Operating system patches* ，用於檢查您的作業系統是否安裝了最新的修補程式，並告訴您如何安裝。

此外，針對 Oracle 精心設計的分析功能會在儲存佈局等組態與其他組態有相依性時發出通知。該通知會告知您需要採取哪些行動來最佳化連結的組態，以便我們提供的所有建議都是最佳的。

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["在 Workload Factory 中實作架構良好的資料庫配置"]



==== Oracle 內部部署資料庫環境成本分析與遷移規劃

Workload Factory for Databases 現在可以分析並協助您規劃將 Oracle 內部部署資料庫移轉至 Amazon FSx for NetApp ONTAP。您可以使用節省成本計算機來估算在雲端中執行 Oracle 內部部署資料庫環境的成本，並檢閱將內部部署資料庫環境移轉至雲端的建議。

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html["探索內部部署資料庫環境的節省方案"]

此次更新將該功能拆分為兩個不同的功能：自動容量管理和容量通知。

自動容量管理功能可自動增加或減少檔案系統容量，以維持最佳效能。新增選項包括增量模式和自適應模式。在增量模式下，系統會在達到預設閾值時以固定量增加容量；在自適應模式下，系統會利用歷史資料來預測和調整容量。

容量通知會在容量調整發生時以及檔案系統達到設定限制時發出警報，以便您在問題出現之前管理儲存。

=== EDA 工作負載



==== EDA 儲存管理概覽儀表板

身為管理數十個 FSx for ONTAP 檔案系統的 IT 管理員、您可以使用 Overview 儀表板來回答有關叢集健全狀況、使用率和磁碟區放置的常見操作問題。

Overview 儀表板包括：

* *叢集健全狀況狀態*：儀表板頂端摘要的資訊，重點說明叢集的效能、容量、可用度與保護及安全性事件。
* *叢集表*：一個可搜尋、可篩選、可分頁的表格，顯示每個叢集的 SSD 容量、容量集區和處理量指標，支援欄自訂和 CSV 匯出。


之前的主儀表板現在可以在 *Project* 標籤中存取。EDA 標籤現在的順序為：Overview、Latency、Project、CI/CD。

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html["深入瞭解總覽儀表板"^]。



==== 自動延遲分析

延遲分析現已包含自動化基礎分析功能。當偵測到延遲事件時，系統會自動分析 ONTAP QoS 延遲中心指標，以識別延遲問題的來源。

基本分析評估 QoS 策略、磁碟效能、網路問題和叢集組態，以便在延遲事件表的「詳細資料」欄中提供即時深入分析。

== 2026 年 2 月 16 日



=== Amazon FSX for NetApp ONTAP 產品



==== 支援儲存 VM 移轉

NetApp Workload Factory 現已支援儲存虛擬機器遷移。此功能允許將 ONTAP 儲存系統資料和配置從本機 ONTAP 系統或第一代 FSx for ONTAP 檔案系統遷移到第二代 FSx for ONTAP 檔案系統。您可以複製儲存虛擬機器的資料和配置設置，以最小的停機時間和對使用者及應用程式的干擾遷移到新的檔案系統。

若要使用此功能，link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["建立複寫關係"]並選擇 *Migration* 作為使用案例。若要完成遷移過程，您必須link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html#cut-over-replication-for-migration-use-cases["切換儲存 VM 及其複寫磁碟區"]立即將資料和儲存 VM 組態設定永久遷移到目標 FSx for ONTAP 檔案系統。

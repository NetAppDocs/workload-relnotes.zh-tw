= 磁碟區資源管理器
:allow-uri-read: 




[[volumes-explorer]]
=== 磁碟區資源管理器

磁碟區資源管理器為您提供 NetApp Workload Factory 中所有磁碟區的簡潔、可搜尋檢視。快速查看哪些磁碟區用於處理區塊或檔案工作負載。查看每個磁碟區的邏輯大小、實體已使用容量以及是否使用了儲存效率功能（壓縮、重複資料刪除和資料精簡）。按名稱、狀態（可用/失敗）、延遲、大小、已使用容量和保護功能進行篩選和排序。



[[enhanced-email-notifications-for-replication-delays]]
=== 增強複製延遲的電子郵件通知

現在，複製延遲的電子郵件通知包含有關受影響關係的更多詳細資訊。此通知可協助您快速識別受影響的關係，並在 Workload Factory 主控台中查看相關資訊。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/track-replication-delays.html["啟用複製延遲通知"]



[[block-device-creation-enhancement]]
=== 區塊裝置建立增強功能

Workload Factory 支援從磁碟區快照建立區塊裝置。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-block-device.html["在 Workload Factory 中從磁碟區快照建立區塊設備"]



[[workload-tagging-for-volumes-and-block-devices]]
=== 磁碟區和區塊設備的工作負載標記

Workload Factory 現在可以識別並標記磁碟區和區塊裝置的主要工作負載或應用程式。您可以指派預先定義的工作負載標籤，例如 Microsoft SQL Server 或 Oracle Database，也可以為應用程式建立自訂標籤。標籤可以套用至個別資源，也可以由連線的儲存資源繼承。這有助於您使儲存設備配置與工作負載需求保持一致、了解事件影響，並更有效地進行疑難排解。
